FEATURES

Double encryption, zero footprint

HTTPS transport wraps XChaCha20-Poly1305 encrypted payloads. Two independent encryption layers with a pre-shared key.

XChaCha20-Poly1305

AEAD encryption with vendored Monocypher extraction. 24-byte random nonces via RtlGenRandom. Wire format: [nonce(24)][mac(16)][ciphertext]. Fresh nonce per message, pre-shared 256-bit key.

HTTPS Transport

Native WinHTTP with TLS on port 443. System proxy support. Chrome User-Agent mimicry. Standard HTTPS that blends with normal browsing.

Jittered Callbacks

Centered jitter via RtlGenRandom. Sleeps between 70-130% of base interval. No predictable beacon pattern for defenders to fingerprint.

Piped Execution

CreateProcess with cmd.exe /c, anonymous pipe capture of stdout+stderr. 64 KB output cap with truncation notification.

ARCHITECTURE

Four-layer design

HTTPS transport wraps XChaCha20-Poly1305 encrypted command payloads. Two components, one build command.

1

WinHTTP Transport

Native Windows TLS via WinHttpOpen / WinHttpSendRequest. Proxy-aware. Self-signed cert bypass for lab use. Standard HTTPS on port 443.

2

XChaCha20-Poly1305

AEAD encryption via vendored Monocypher. Wire format: [nonce(24)][mac(16)][ct]. Fresh 24-byte random nonce per message via RtlGenRandom.

3

Beacon Protocol

POST /poll with encrypted beacon ID. POST /result with encrypted output. HTTP 200 = command queued, HTTP 204 = sleep.

4

Python Listener

HTTPS server with interactive prompt. Thread-safe command queue. PyNaCl for matching XChaCha20-Poly1305 decrypt/encrypt. Auto-generated TLS certs.

GETTING STARTED

Build and deploy

Cross-compile from Linux with MinGW-w64. One command generates everything.

build.sh
$ git clone https://github.com/Real-Fruit-Snacks/Dew.git
$ cd Dew

# Build with auto-generated PSK
$ ./build.sh 10.10.14.1 443

[*] Compiled dew.exe (37,888 bytes)

# Or with your own key
$ ./build.sh 10.10.14.1 443 <64-hex>

DETECTION AWARENESS

Know the boundaries

Hidden From

  • Network content inspection — double-encrypted (TLS + AEAD)
  • Beacon fingerprinting — randomized jitter intervals
  • Protocol analysis — standard HTTPS on port 443
  • User-Agent filtering — mimics Chrome browser
  • Payload inspection — encrypted binary POST bodies

Visible To

  • Process monitoring — WinHTTP connections from dew.exe
  • ETW / kernel callbacks — process creation events
  • TLS inspection — self-signed certificate detection
  • Memory forensics — PSK and code in process memory
  • Endpoint protection — static and behavioral analysis
